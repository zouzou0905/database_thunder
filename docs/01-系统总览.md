# 01-系统总览

## 项目定位

本项目是面向运营的 Amazon ABA / SellerSprite 关键词趋势数据库和运营工作台。系统负责把 CSV 关键词月度数据导入 PostgreSQL，进行清洗、归一化、趋势计算、机会池缓存、横向对比预计算，并通过前端和管理后台提供查询、筛选、导出、用户管理和错误排查能力。

当前系统只处理关键词维度数据，不包含产品价格、利润、ASIN、竞品或商品供应链维度。如果后续扩展到商品维度，应作为独立增强模块设计。

## 系统组成

```text
CSV 原始文件
  -> Python 导入、清洗、去重
  -> PostgreSQL
  -> 趋势计算、机会池缓存、横向对比快照、节日标签
  -> FastAPI JSON API
  -> React 运营工作台

PostgreSQL
  -> Admin FastAPI + Jinja2 SSR
  -> 管理后台
```

运营前端用于日常筛选、对比、导出和词库维护。管理后台用于管理员登录、用户 CRUD、错误日志查看和数据库表状态查看。

## 预计算原则

系统采用“预计算优先，实时计算兜底”的策略。

```text
keyword_selection_candidates_monthly   机会池主要查询来源
keyword_compare_snapshot               全量区间横向对比快照
keyword_compare_range_cache            常用日期范围横向对比缓存
keyword_compare_scenario_snapshot      节日/场景类横向对比快照
keyword_compare_interval_snapshot      固定区间横向对比快照
keyword_holiday_tags                   节日标签缓存
v_mb_product_selection_candidates       机会池缓存缺失时兜底视图
```

API 会返回 `cached` 字段。`cached=false` 表示正在使用实时视图或实时计算路径，页面可能变慢，应检查缓存是否需要刷新。

## 当前边界

```text
不把 Metabase 作为运营主系统
不允许前端直接连接 PostgreSQL
不让运营人员直接操作全量数据库
不在 API 启动时自动重建缓存
不在当前阶段引入 ASIN、利润、竞品维度
```

缓存、快照和节日标签需要通过脚本或后台接口有意刷新，避免服务启动时触发重型数据库任务。
