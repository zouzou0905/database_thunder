# 01-系统总览

## 项目定位

本项目是一个面向运营的关键词机会筛选系统。它不是单纯的 CSV 存储工具，也不是通用 BI 看板。

当前核心目标：

```text
基于 2025-04 到 2026-04 的 Amazon ABA / SellerSprite 关键词月度数据，
帮助运营筛出值得进一步调研的关键词，
提高选品方向发现数量和筛选效率。
```

当前系统只依赖关键词维度数据，不依赖产品售价、利润、完整 ASIN、竞品商品详情。后续如果补充产品维度，可以作为增强模块接入。

## 当前技术方向

```text
开发机验证
-> 真实中心服务器部署
-> PostgreSQL 数据库
-> Python 数据导入与计算脚本
-> FastAPI 后端
-> React Web 前端
-> Electron 软件壳后置
```

当前这台电脑只是开发机/验证机，不是最终中心服务器。代码完成后会迁移到真正的中心服务器。

运营正式使用时，通过浏览器访问中心服务器上的 Web 页面。Electron 只是后续软件壳，本质仍然访问同一个 Web 后台。

## 系统架构

```text
CSV 原始文件
  -> Python 导入、清洗、去重
  -> PostgreSQL
  -> 趋势计算、机会评分、横向对比快照
  -> FastAPI API
  -> React 运营工作台
  -> 运营浏览器访问
```

Metabase 只作为辅助分析工具，不作为运营主入口。

## 当前核心模块

```text
数据导入
趋势计算
关键词机会池
横向对比
收藏、状态、备注
禁用词维护
Excel 导出
```

## 性能原则

系统采用“预计算优先，实时计算兜底”：

```text
keyword_selection_candidates_monthly   关键词机会池主查询来源
keyword_compare_snapshot_monthly       横向对比快照
v_mb_product_selection_candidates       缓存缺失时兜底
```

API 会返回 `cached` 字段。`cached=false` 表示正在回退实时视图，页面会提示查询较慢，应补跑缓存刷新任务。

## 当前不做的事

```text
不把 Metabase 当运营主系统
不让前端直接连接 PostgreSQL
不在运营电脑保存全量数据库
不把 Electron 放在第一阶段
不依赖 ASIN、利润、竞品维度做第一阶段判断
```
