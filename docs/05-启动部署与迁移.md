# 05-启动部署与迁移

## 启动顺序

```text
PostgreSQL
-> FastAPI 后端 API，端口 8001
-> React 运营前端，端口 8000
-> Admin 管理后台，端口 8002
```

当前项目目录：

```text
D:\database\database_thunder-main
```

当前局域网访问 IP：

```text
192.168.0.135
```

## 启动后端 API

```powershell
cd D:\database\database_thunder-main
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe -m uvicorn app.main:app --app-dir backend --host 0.0.0.0 --port 8001
```

启动成功会看到：

```text
Uvicorn running on http://0.0.0.0:8001
Application startup complete.
```

后端检查地址：

```text
http://127.0.0.1:8001/api/health
http://127.0.0.1:8001/docs
http://192.168.0.135:8001/api/health
```

直接打开 `http://127.0.0.1:8001/` 返回 404 是正常的，后端只提供 API。

## 后端 .env 注意事项

后端启动命令使用了 `--app-dir backend`，运行时 CWD 会变为 `backend/`。因此：

```text
项目根目录 .env
backend/.env
```

两份文件都需要保持关键配置一致，尤其是：

```text
DATABASE_URL
JWT_SECRET
DEFAULT_MARKETPLACE
CORS_ORIGINS
ADMIN_SESSION_SECRET
```

当前数据库连接：

```text
postgresql://keyword_user:<password>@localhost:5432/keyword_trends
```

## 启动运营前端

```powershell
cd D:\database\database_thunder-main\frontend
$env:VITE_API_BASE_URL = "http://192.168.0.135:8001/api"
npm run dev
```

前端地址：

```text
http://127.0.0.1:8000
http://192.168.0.135:8000
```

## 启动管理后台

推荐使用脚本：

```powershell
cd D:\database\database_thunder-main
.\admin\start_admin.ps1
```

或手动启动：

```powershell
cd D:\database\database_thunder-main
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend;admin"
.\.venv\Scripts\python.exe -m uvicorn admin.app.main:app --host 0.0.0.0 --port 8002
```

管理后台地址：

```text
http://127.0.0.1:8002/admin
http://192.168.0.135:8002/admin
```

登录用户必须是 `role=admin` 且未停用。

## 局域网访问要求

```text
前端监听 0.0.0.0:8000
后端监听 0.0.0.0:8001
管理后台监听 0.0.0.0:8002
Windows 防火墙允许 8000、8001、8002
前端 VITE_API_BASE_URL 指向可访问的后端地址
```

## 服务不会自动重建缓存

FastAPI 启动只启动 API，不会自动刷新趋势、机会池、横向对比快照或节日标签。

导入新月份、修改词库或 PostgreSQL 重启后，需要按需运行：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts\calculate_trends.py --all --marketplace UK
```

节日词库修改后，还需要刷新节日标签。

## 停止服务

在对应 PowerShell 窗口按：

```text
Ctrl + C
```

端口被占用时检查：

```powershell
netstat -ano | findstr :8000
netstat -ano | findstr :8001
netstat -ano | findstr :8002
```

必要时结束指定 PID：

```powershell
taskkill /F /PID <PID>
```
