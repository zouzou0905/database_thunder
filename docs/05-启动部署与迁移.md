# 05-启动部署与迁移

## 本机启动顺序

```text
PostgreSQL
-> FastAPI 后端，端口 8001
-> React 前端，端口 8000
```

## 启动后端

```powershell
cd F:\database
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
.\.venv\Scripts\Activate.ps1
.\backend\start_api.ps1
```

看到下面内容表示成功：

```text
Uvicorn running on http://0.0.0.0:8001
Application startup complete.
```

后端地址：

```text
http://127.0.0.1:8001/api
http://127.0.0.1:8001/docs
http://127.0.0.1:8001/api/health
```

直接打开 `http://127.0.0.1:8001/` 出现 404 是正常的，因为后端不是网页首页。

## 启动前端

```powershell
cd F:\database\frontend
npm.cmd run dev
```

前端地址：

```text
http://127.0.0.1:8000
```

当前开发账号：

```text
账号：使用本地创建的账号
密码：使用本地创建的密码
```

## 局域网访问

其他电脑访问当前开发机时，用开发机局域网 IP：

```text
http://192.168.0.127:8000
```

需要确保：

```text
前端监听 0.0.0.0:8000
后端监听 0.0.0.0:8001
Windows 防火墙放行 8000 和 8001
前端 API 地址指向可访问的后端地址
```

## 性能任务不会随后端启动自动运行

FastAPI 启动只启动 API，不会自动重建缓存。

导入新月份、补导旧月份、修改评分规则后，需要单独运行：

```powershell
python scripts\calculate_trends.py --analysis-month 2026-04 --marketplace UK
```

## 停止服务

在前端或后端 PowerShell 窗口按：

```text
Ctrl + C
```

端口被占用时检查：

```powershell
netstat -ano | findstr :8000
netstat -ano | findstr :8001
```

## 迁移到中心服务器

当前电脑不是最终中心服务器。迁移时需要确认：

```text
PostgreSQL 已安装并恢复数据
后端 .env 数据库连接正确
前端 VITE_API_BASE_URL 指向中心服务器后端
防火墙开放 8000 和 8001
导入脚本和计算脚本能在服务器运行
缓存刷新任务有明确执行方式
备份策略已建立
```

第一阶段可以手动启动。稳定后再配置为系统服务或 Docker Compose。
