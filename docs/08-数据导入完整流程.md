# 08-数据导入完整流程

每次导入新月份数据后，按顺序执行以下步骤。日常速查见 [04-数据导入与刷新手册.md](04-数据导入与刷新手册.md)。

## 前置条件

从项目根目录执行：

```powershell
cd D:\database\database_thunder-main
.\.venv\Scripts\Activate.ps1
```

首次部署或 SQL 文件有新增时，初始化数据库结构：

```powershell
python scripts/init_db.py
```

当前初始化脚本会加载：

```text
sql/schema/001_core.sql 至 013_keyword_compare_interval_snapshot.sql
sql/indexes/001_core_indexes.sql 至 004_compare_interval_indexes.sql
sql/views/001_core_views.sql 至 003_product_selection_cached_view.sql
```

关键缓存和快照表：

```text
keyword_selection_candidates_monthly
keyword_compare_snapshot
keyword_compare_range_cache
keyword_compare_scenario_snapshot
keyword_compare_interval_snapshot
keyword_holiday_tags
```

## 第 1 步：CSV 预检

检查 CSV 文件格式、行数、重复关键词和缺失字段。

```powershell
python scripts/inspect_csv.py `
  --input "D:\PostgreSQL\data\aba数据\YYYY-MM" `
  --output data/reports/YYYY-MM-inspect.json
```

查看报告确认无异常后再导入。

## 第 2 步：导入 CSV

```powershell
python scripts/import_csv.py `
  --month YYYY-MM `
  --marketplace UK `
  --input "D:\PostgreSQL\data\aba数据\YYYY-MM"
```

导入链路：

```text
CSV
  -> raw_aba_rows
  -> keywords
  -> keyword_monthly_metrics
```

同一文件通过 SHA256 自动跳过，避免重复导入。如确认需要覆盖已有月份数据：

```powershell
python scripts/import_csv.py `
  --month YYYY-MM `
  --marketplace UK `
  --input "D:\PostgreSQL\data\aba数据\YYYY-MM" `
  --force
```

## 第 3 步：刷新趋势、机会池和横向对比

完整刷新所有已导入月份：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts/calculate_trends.py --all --marketplace UK
```

单月刷新：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts/calculate_trends.py --analysis-month YYYY-MM --marketplace UK
```

`calculate_trends.py` 当前负责：

| 子步骤 | 输出 | 说明 |
| --- | --- | --- |
| 趋势计算 | `keyword_monthly_trends` | 环比、同比、趋势标签等 |
| 运营评分 | `keyword_ops_monthly` | 机会池评分和推荐动作 |
| 机会池缓存 | `keyword_selection_candidates_monthly` | 机会池主要查询来源 |
| 全量对比快照 | `keyword_compare_snapshot` | 横向对比全量区间来源 |
| 常用区间预热 | `keyword_compare_range_cache` | 自定义区间缓存，UNLOGGED |

如只想计算趋势和运营评分，不刷新机会池缓存、快照和预热：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts/calculate_trends.py --all --marketplace UK --skip-cache-refresh
```

## 第 4 步：按需刷新节日标签

如果本次导入涉及节日窗口月份，或者节日词库有改动，需要刷新 `keyword_holiday_tags`。

推荐通过前端“节日词库”页面触发刷新；也可以通过 API：

```powershell
Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:8001/api/holiday-tags/refresh?marketplace=UK"
```

节日标签依赖：

```text
holiday_events
holiday_terms
keyword_monthly_metrics
keywords
```

## 第 5 步：按需刷新固定区间和场景快照

这一步不是每次导入都必须执行。只有当你需要让横向对比优先命中特定固定区间或场景快照时再运行。

固定区间：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts\refresh_compare_intervals.py --marketplace UK
```

场景快照：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts\refresh_compare_scenarios.py --marketplace UK
```

## 常见问题

### 横向对比全量区间很慢

检查 `keyword_compare_snapshot` 是否为空，或 `calculate_trends.py --all --marketplace UK` 是否跑完。

### 自定义区间首次查询很慢

该区间未命中 `keyword_compare_range_cache`、`keyword_compare_interval_snapshot` 或 `keyword_compare_scenario_snapshot`，后端会走 live CTE 兜底。常用范围应提前预热或做固定区间/场景快照。

### PostgreSQL 重启后横向对比变慢

`keyword_compare_range_cache` 是 UNLOGGED 表，PostgreSQL 重启后内容会丢失。重新执行：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts\prewarm_compare_range_cache.py --marketplace UK
```

或完整刷新：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts\calculate_trends.py --all --marketplace UK
```

### 机会池显示 cached=false

说明 `keyword_selection_candidates_monthly` 对应月份缓存缺失，刷新机会池缓存：

```powershell
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts\refresh_product_selection_cache.py --analysis-month YYYY-MM --marketplace UK
```

### 节日筛选没有变化

检查 `keyword_holiday_tags` 是否已刷新。词库改动不会自动重算历史标签。
