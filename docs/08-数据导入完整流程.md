每次导入新月份数据后，按顺序执行以下步骤。

---

## 数据导入流程（每月执行）

### 第 1 步：CSV 预检

检查 CSV 文件格式、行数、重复关键词、缺失字段。

```powershell
.\.venv\Scripts\Activate.ps1
python scripts/inspect_csv.py `
  --input "D:\PostgreSQL\data\aba数据\YYYY-MM" `
  --output data/reports/YYYY-MM-inspect.json
```

查看报告，确认无异常后继续。

### 第 2 步：导入 CSV

将 CSV 数据写入 `raw_aba_rows` → `keywords` → `keyword_monthly_metrics`。

```powershell
.\.venv\Scripts\Activate.ps1
python scripts/import_csv.py `
  --month YYYY-MM `
  --marketplace UK `
  --input "D:\PostgreSQL\data\aba数据\YYYY-MM"
```

- 如需强制重新导入（覆盖已有数据）：加 `--force`
- 相同文件（SHA256）自动跳过，不会重复导入

### 第 3 步：计算趋势 + 刷新缓存 + 预热区间

**这是最关键的一步**，必须跑完。单一月份：

```powershell
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts/calculate_trends.py `
  --analysis-month YYYY-MM `
  --marketplace UK
```

重算所有月份（首次导入或修复数据时）：

```powershell
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts/calculate_trends.py `
  --all `
  --marketplace UK
```

这一步会自动完成：

| 子步骤 | 说明 | 耗时 |
|---|---|---|
| 趋势表 | 计算环比、同比、趋势标签 | ~1s/月 |
| Ops 表 | 计算运营评分、推荐动作 | ~1s/月 |
| 选品缓存 | 刷新 `keyword_selection_candidates_monthly` | ~5s/月 |
| 全周期快照 | 刷新 `keyword_compare_snapshot` | ~30s |
| **区间预热** | 预热 15 个常用区间到 `keyword_compare_range_cache` | **~5 分钟** |

区间预热覆盖：13 个单月 + 近3月 + 近6月。预热后横向对比所有常用区间查询均为秒出。

如需跳过缓存刷新（仅算趋势，不更新快照/预热）：

```powershell
.\.venv\Scripts\python.exe scripts/calculate_trends.py --all --marketplace UK --skip-cache-refresh
```

### 验证区间缓存已预热

```powershell
.\.venv\Scripts\python.exe -c "
import psycopg
conn = psycopg.connect('postgresql://keyword_user:keyword_user_123456@localhost:5432/keyword_trends')
cur = conn.cursor()
cur.execute('''
  SELECT start_month, end_month, COUNT(*) as cnt
  FROM keyword_compare_range_cache
  GROUP BY start_month, end_month
  ORDER BY start_month
''')
for r in cur.fetchall():
    print(f'{r[0]} -> {r[1]}: {r[2]:,} 条')
conn.close()
"
```

### 验证横向对比功能

打开前端 → 横向对比 → 选择不同起止月份 → 确认数据秒出且增长率正确。

## 常见问题

### Q: 首次查询新区间很慢（>1 分钟）

A: 该区间未被预热。`calculate_trends.py` 跑完后会自动预热 15 个常用区间。如果用户选了非预热区间（如 2025-07~2025-09），首次会自动建缓存（可能 30-120 秒），后续同区间秒出。

### Q: PostgreSQL 重启后查询变慢

A: `keyword_compare_range_cache` 是 UNLOGGED 表，PostgreSQL 重启后数据丢失。需重新预热：

```powershell
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts/calculate_trends.py --all --marketplace UK
```

### Q: 导入新月份后横向对比没有新数据

A: 检查 `calculate_trends.py` 是否跑完。新月份的数据需要 Step 3（全周期快照）+ Step 4（区间预热）才能出现在横向对比中。用上面的验证命令确认 `keyword_compare_range_cache` 中有新月份的记录。

### Q: 机会池报 "cached: false"

A: 选品缓存表无数据。单独刷新选品缓存：

```powershell
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe scripts/refresh_product_selection_cache.py --month YYYY-MM --marketplace UK
```
