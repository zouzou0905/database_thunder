# 02-数据模型与计算链路

## 数据库分层

当前 PostgreSQL 数据库按五层理解：

```text
导入管理层
原始与标准化层
分析与缓存层
运营动作层
管理与审计层
```

## 导入管理层

```text
import_batches   每次导入批次
source_files     每个 CSV 文件记录，使用 SHA256 文件 hash 防重复导入
```

## 原始与标准化层

```text
raw_aba_rows             CSV 原始行
keywords                 关键词主表，保存归一化关键词和 keyword_hash
keyword_monthly_metrics  关键词月度指标事实表
```

`keyword_monthly_metrics` 是趋势、机会池、横向对比和节日标签计算的核心底表。

## 分析与缓存层

```text
keyword_monthly_trends                  月度趋势结果
keyword_ops_monthly                     运营评分和推荐理由
keyword_selection_candidates_monthly     机会池月度缓存
keyword_compare_snapshot                全量区间横向对比快照
keyword_compare_snapshot_stats          横向对比快照统计
keyword_compare_range_cache             自定义日期范围横向对比缓存，UNLOGGED
keyword_compare_scenario_snapshot       节日/场景类横向对比快照
keyword_compare_interval_snapshot       固定区间横向对比快照
keyword_holiday_tags                    节日标签缓存
v_mb_product_selection_candidates       机会池实时兜底视图
```

`keyword_compare_range_cache` 是 UNLOGGED 表，PostgreSQL 重启后数据会丢失，需要重新预热。

## 节日词库层

```text
holiday_events  节日定义，例如 halloween、christmas
holiday_terms   节日命中词典，支持 phrase 和 word 匹配
```

节日标签由 `backend/app/services/holiday_tags.py` 计算，写入 `keyword_holiday_tags`。词库变更后必须重新刷新节日标签，否则横向对比里的节日筛选和标签展示仍会使用旧缓存。

## 运营动作层

```text
app_users                       应用用户
keyword_selection_states        收藏、状态等用户维度操作
keyword_selection_notes         关键词备注
keyword_selection_exclusions    禁用词、品牌词、排除规则
shared_clipboard_items          共享粘贴板
```

## 管理与审计层

```text
error_logs      后端和管理后台错误日志
login_history   登录历史
```

管理后台用户删除会清理该用户相关的状态、备注和粘贴板记录，并把登录历史中的 `user_id` 置空。

## 月份口径

系统中的“月份”统一使用 `YYYY-MM` 或数据库中的月初日期。

例如分析月份为 `2026-04`：

```text
当前月 = 2026-04
上月 = 2026-03
近 3 月 = 2026-02 至 2026-04，具体口径由接口/脚本参数决定
去年同月 = 2025-04
```

前端传入 `YYYY-MM`，后端统一转为数据库可比较的月份日期。

## 主计算链路

```text
CSV 文件
  -> inspect_csv.py
  -> import_csv.py
  -> raw_aba_rows
  -> keywords
  -> keyword_monthly_metrics
  -> calculate_trends.py
  -> keyword_monthly_trends
  -> keyword_ops_monthly
  -> keyword_selection_candidates_monthly
  -> keyword_compare_snapshot
  -> keyword_compare_range_cache
```

`calculate_trends.py --all --marketplace UK` 是导入后最常用的完整刷新命令。它会刷新趋势、运营评分、机会池缓存、全量横向对比快照，并预热常用横向对比日期范围。

场景快照和固定区间快照另有独立脚本：

```text
scripts/refresh_compare_scenarios.py
scripts/refresh_compare_intervals.py
```

## 横向对比数据源

横向对比不是每次都直接扫描大事实表，而是按优先级选择数据源：

```text
全量区间             -> keyword_compare_snapshot
固定区间             -> keyword_compare_interval_snapshot
节日/场景             -> keyword_compare_scenario_snapshot
已预热的自定义区间     -> keyword_compare_range_cache
未命中的自定义区间     -> live CTE 实时计算并写入缓存
```

实时 CTE 是兜底路径，可能扫描 `keyword_monthly_metrics` 并执行聚合、回归、同比/环比计算。常用筛选范围应尽量走快照或缓存。

## 横向对比主要计算字段

```text
末月搜索量
搜索量增量
增长率
环比变化和环比率
同比变化和同比率
出现月数
平均搜索量
PPC
SPR
趋势类型
节日标签
月度搜索量明细
历史排名参考
```

趋势类型为：

```text
rising      上升型
falling     下降型
stable      常年稳定型
seasonal    季节型
volatile    波动型
```

## 查询要求

```text
所有列表接口必须分页
高频页面优先查询缓存表或快照表
实时视图和 live CTE 只用于兜底
缓存缺失时前端需要给出提示
词库、缓存、快照刷新必须显式执行，不在 API 启动时自动执行
```
