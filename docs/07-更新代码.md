# 07-更新代码

从 GitHub 上游仓库拉取最新代码，同时保护本地配置和数据库数据。

## 更新步骤

```powershell
cd D:\database\database_thunder-main
git pull origin main --allow-unrelated-histories
```

如果出现冲突，优先保留本地的 `.env`：

```powershell
git checkout --ours .env
```

## 更新后检查

如果 `requirements.txt` 有变化，重新安装依赖：

```powershell
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

如果 `sql/schema/` 或 `sql/indexes/` 目录有新文件，运行数据库初始化（已存在的表不会受影响）：

```powershell
python scripts/init_db.py
```

如果 `frontend/package.json` 有变化：

```powershell
cd frontend
npm install
```

## 重启服务

```powershell
# 停旧进程
netstat -ano | findstr :8000
netstat -ano | findstr :8001
taskkill /F /PID <PID>

# 启动后端
cd D:\database\database_thunder-main
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend"
.\.venv\Scripts\python.exe -m uvicorn app.main:app --app-dir backend --host 0.0.0.0 --port 8001

# 启动前端（新开 PowerShell）
cd D:\database\database_thunder-main\frontend
$env:VITE_API_BASE_URL = "http://192.168.0.135:8001/api"
npm run dev
```

## 不会受影响的内容

- `.env` — 数据库连接、CORS 配置
- `data/` — 本地数据文件
- `.venv/` — Python 虚拟环境
- PostgreSQL 数据库 — 所有已导入数据和用户账号

## 注意事项

- 首次 `git pull` 需要 `--allow-unrelated-histories`，因为本地仓库和远程仓库没有共同提交历史
- 后续更新不用加这个参数
- 如果原作者删除了仓库或设为私有，连接会失效
