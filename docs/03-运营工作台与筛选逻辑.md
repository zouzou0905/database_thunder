# 03-运营工作台与筛选逻辑

## 工作台定位

运营工作台用于筛选关键词机会，不用于直接给出最终产品结论。

系统负责：

```text
减少无效词
降低误判
把关键词分流到不同机会池
保留运营判断动作
支持导出和复盘
```

运营负责：

```text
结合产品可开发性、供应链、利润、合规、竞品情况做最终判断
```

## 主要页面

```text
机会池
横向对比
收藏夹
禁用词
```

## 机会池分流

当前机会池不再追求一个黑盒“高分榜”，而是把关键词分流给运营判断：

```text
全量词库
候选词库
核心选品机会
上升趋势机会
稳定需求机会
新词观察
低竞争机会
高需求谨慎池
待人工判断
```

其中“高需求谨慎池”不是排除池。高搜索量词可能有价值，但通常竞争更强，所以需要单独判断。

## 趋势标签

趋势标签必须同时看搜索量和排名，避免“搜索量涨但排名跌”也被简单叫做上升。

当前推荐标签：

```text
上升机会
新出现词
稳定词
需求升但排名降
排名升但需求降
下降风险
```

前端下拉框只展示这些对运营有意义的标签，复杂内部条件不直接塞进筛选框。

## 评分原则

评分用于排序和分层，不用于替代运营判断。

评分应避免：

```text
所有词都高分
月份越近天然越高
只因搜索量大就高分
搜索量涨但排名恶化还被当成普通上升
```

前端需要展示必要解释，例如：

```text
推荐原因
排除原因
降级原因
状态
收藏
备注
```

## 禁用词与收藏

运营可对关键词执行：

```text
收藏
取消收藏
设置状态
添加备注
加入禁用词
启用或停用禁用规则
```

禁用词不能只在机会池页面生效，横向对比页面也应支持收藏、状态和禁用词动作。

## 横向对比模块

横向对比单独成模块，用来回答：

```text
同一个关键词在多个月份里，是上升、下降、稳定，还是偶然出现？
```

默认筛选建议：

```text
起始月份 = 数据库最早月份
结束月份 = 数据库最新月份
站点 = UK
至少出现 3 个月
末月搜索量有下限
按末月搜索量或增长率排序
```

横向对比不使用机会池评分作为默认排序依据，避免把两个不同逻辑混在一起。

## 前端交互要求

```text
筛选条件要可见、可复选、可重置
表格必须分页
可以跳转页码
切换页面后尽量保留原筛选和页码
查询数据库时显示加载提示
当前选中的模块必须有清晰高亮
```
