# 03-运营工作台与筛选逻辑

## 工作台定位

运营工作台用于筛选关键词机会、对比关键词趋势、维护禁用词和节日词库，并完成数据导出。系统不直接给出选品结论，只把关键词数据整理成更适合运营判断的结构。

运营仍需要结合：

```text
产品可供应性
合规风险
竞品强度
季节性
内容和广告投放策略
人工判断
```

## 当前页面

前端当前由 `frontend/src/components/Sidebar.tsx` 控制 7 个主模块：

```text
机会池
我的收藏
横向对比
禁用词
数据导出
共享粘贴板
节日词库
```

`App.tsx` 负责状态、筛选条件、加载、导出和各模块切换；具体表格和表单拆分在 `frontend/src/components/` 下。

## 机会池

机会池主要查询 `keyword_selection_candidates_monthly`。缓存缺失时可以兜底到 `v_mb_product_selection_candidates`，接口会通过 `cached` 字段提示数据来源。

机会池用于回答：

```text
哪些关键词在当前分析月值得进一步查看
哪些词增长明显
哪些词稳定、有需求
哪些词竞争压力相对可控
哪些词需要排除或备注
```

机会池支持收藏、状态、备注和禁用词动作。当前不再在机会池页面单独承担数据导出入口，导出统一由“数据导出”模块完成。

## 我的收藏

我的收藏复用机会池列表逻辑，但只展示当前用户收藏过的关键词。收藏状态来自 `keyword_selection_states`。

## 横向对比

横向对比用于比较同一个关键词在多个已导入月份内的表现，重点看：

```text
末月搜索量
搜索量增量
增长率
环比
同比
排名变化
出现月数
平均搜索量
PPC
SPR
趋势类型
节日标签
月度搜索量明细
```

默认数据源优先走快照和缓存：

```text
全量区间 -> keyword_compare_snapshot
固定区间 -> keyword_compare_interval_snapshot
节日/场景 -> keyword_compare_scenario_snapshot
自定义常用区间 -> keyword_compare_range_cache
兜底 -> live CTE
```

横向对比支持节日筛选。节日筛选依赖 `keyword_holiday_tags`，词库变更后必须刷新节日标签。

## 禁用词

禁用词用于维护品牌词、无关词、风险词和其他排除规则。禁用词规则影响机会池和横向对比中的运营动作，但不删除底层关键词数据。

## 数据导出

数据导出是当前唯一的导出入口。可选择：

```text
数据来源：机会池 / 横向对比
文件名称
文件格式
导出条数
导出字段
```

导出结果区域显示：

```text
总条数：当前筛选条件下数据库可导出的总记录数
预计导出条数：运营本次选择导出的记录数
自动分批：超过 20,000 条时按批次导出
```

运营可以手动输入任意导出条数，也可以使用快捷选项。若输入条数大于总条数，实际导出以总条数为上限。

## 共享粘贴板

共享粘贴板用于在本机、远程桌面、运营和开发之间临时同步文本、脚本片段和说明。它是辅助工具，不参与关键词评分。

## 节日词库

节日词库维护 `holiday_events` 和 `holiday_terms`。当前支持 Halloween 和 Christmas 等节日词典，并可继续扩展。

节日标签计算逻辑：

```text
关键词或翻译命中启用中的 holiday_terms
phrase 类型使用包含匹配
word 类型使用单词边界匹配
命中后检查节日窗口内趋势是否符合增长要求
结果写入 keyword_holiday_tags
```

词库新增、删除、启用或停用后，前端会提示缓存可能过期。需要通过刷新节日标签接口或维护脚本重建 `keyword_holiday_tags`。

## 前端交互要求

```text
筛选栏必须清晰可见
列表必须分页
支持页码跳转
切换页面时尽量保留各自筛选状态
查询中显示 loading
错误要展示给运营人员
长列表导出必须走分批逻辑
节日词库变更后要提示缓存状态
```
