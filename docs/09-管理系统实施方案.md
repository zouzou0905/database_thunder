# 09-管理系统实施方案

> 当前版本：2026-07-09  
> 管理后台地址：`http://192.168.0.135:8002/admin`

## 定位

管理后台是独立于运营前端的管理员系统，用于：

```text
管理员登录和会话管理
用户创建、编辑、停用、删除
错误日志查看和处理
数据库表状态查看
```

它和后端 API、运营前端共用同一个 PostgreSQL 数据库，但使用独立的 FastAPI + Jinja2 SSR 应用，不复用 React 前端。

## 架构

```text
运营前端 React SPA      后端 FastAPI JSON API       管理后台 FastAPI SSR
Port 8000              Port 8001                  Port 8002
     |                      |                          |
     +------ 运营用户 ------+          管理员 ----------+
                                                    |
                                              Jinja2 模板
                                              Session cookie
                                              PostgreSQL
```

管理后台使用 Starlette `SessionMiddleware` 保存登录态，session cookie 有签名保护。每次访问受保护页面时都会重新查询数据库，确认当前用户仍是启用状态且 `role=admin`。

## 启动方式

推荐：

```powershell
cd D:\database\database_thunder-main
.\admin\start_admin.ps1
```

手动：

```powershell
cd D:\database\database_thunder-main
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "backend;admin"
.\.venv\Scripts\python.exe -m uvicorn admin.app.main:app --host 0.0.0.0 --port 8002
```

注意：`PYTHONPATH` 必须是 `backend;admin`，且 `backend` 在前，避免 `app` 包名冲突。

## 路由

当前 `admin/app/main.py` 挂载：

```text
auth
dashboard
users
database_tables
error_logs
```

### 认证

```text
GET  /admin/login
POST /admin/login
POST /admin/logout
```

登录要求：

```text
用户存在
is_active = true
role = admin
密码校验通过
```

### 仪表盘

```text
GET /admin/
```

展示系统概览和管理入口。

### 用户管理

```text
GET  /admin/users
GET  /admin/users/create
POST /admin/users
GET  /admin/users/{user_id}/edit
POST /admin/users/{user_id}
POST /admin/users/{user_id}/toggle
POST /admin/users/{user_id}/delete
```

当前没有独立的 `reset-password` 路由。管理员如需修改用户密码，在编辑用户页面填写新密码并保存。

删除用户规则：

```text
不能删除当前登录管理员自己
删除 app_users 记录
清理该用户的 keyword_selection_states
清理该用户的 keyword_selection_notes
清理该用户的 shared_clipboard_items
login_history.user_id 置为 NULL
```

停用用户只改变 `is_active`，不会删除历史数据。

### 数据库表

```text
GET /admin/database-tables
```

该页面用于查看当前数据库中存在的表，帮助确认初始化、导入、缓存和快照状态。它是只读页面，不执行建表、删表或清空数据操作。

### 错误日志

```text
GET  /admin/error-logs
GET  /admin/error-logs/{error_id}
POST /admin/error-logs/{error_id}/resolve
```

错误日志来自 `error_logs` 表。后端的 `ErrorCaptureMiddleware` 会捕获未处理异常并写入该表，管理后台可查看详情和标记处理。

## 数据库连接

管理后台使用独立连接池，配置在 `admin/app/pool.py`。

```text
min_size = 2
max_size = 10
```

连接信息来自 `.env`：

```text
DATABASE_URL
ADMIN_SESSION_SECRET
```

管理后台关闭时会执行连接池 shutdown handler，释放连接。

## 用户密码

用户密码使用后端统一的安全逻辑：

```text
backend/app/core/security.py
PBKDF2-SHA256
```

创建用户脚本：

```powershell
python scripts/create_app_user.py --account admin --display-name "管理员" --role admin --password <password>
```

管理后台创建/编辑用户时使用 JSON POST，避免旧版本 Starlette + python-multipart 在中文表单场景下出现编码问题。

## 主要数据表

```text
app_users
login_history
error_logs
keyword_selection_states
keyword_selection_notes
shared_clipboard_items
```

数据库表查看页面会读取 PostgreSQL 元数据，不直接依赖固定表清单。

## 权限边界

```text
管理后台只允许 admin 角色访问
普通运营用户不能进入管理后台
删除用户是物理删除，操作前需要人工确认
数据库表页面只读
错误日志处理只改变处理状态，不删除日志
```

## 与运营前端的关系

管理后台修改用户后，运营前端不会通过 websocket 实时同步。实际效果如下：

```text
停用用户：用户下一次请求或重新登录时失效
修改密码：用户下一次登录使用新密码
删除用户：用户记录被删除，相关操作数据被清理
```

如果某个用户已经登录，后端接口会在认证时继续校验用户状态；被停用或删除后不应继续拥有有效业务访问能力。

## 维护注意事项

```text
不要把管理后台和运营前端混成一个 React 应用
不要让管理后台直接执行重型缓存刷新
不要在数据库表页面加入破坏性操作
新增管理后台页面时同步更新 base.html 导航
新增管理后台路由时同步更新本文件
```
