# 06-开发规范与前端设计

## 技术栈

```text
数据库: PostgreSQL
数据任务: Python
后端 API: Python + FastAPI + psycopg
前端: React + TypeScript + Vite
桌面壳: Electron + TypeScript，后续可选
```

不要同时引入多套同类技术：

```text
后端不混用 FastAPI 和 Express
前端不混用 React 和 Vue
数据清洗不混用 Python 和 Node
桌面端不直接承载业务数据逻辑
```

## 依赖管理

Python 依赖以项目根目录的 `requirements.txt` 为准。

推荐使用项目虚拟环境：

```text
F:\database\.venv
```

迁移项目时，不提交 `.venv`，在新机器重新安装依赖。

## 后端规范

```text
所有前端业务操作必须经过 FastAPI
前端不能直接连接 PostgreSQL
数据库连接使用连接池
普通接口设置 statement_timeout
重计算任务用脚本执行，不放在 API 启动流程里
接口必须分页，不能返回全量大表
```

## SQL 规范

```text
表结构、索引、视图必须落在 sql/ 目录
不要只在 pgAdmin 手工改结构
常用 WHERE 和 ORDER BY 字段要建索引
多人高频页面优先查缓存表或快照表
实时视图只能兜底或用于验证
```

## 前端设计原则

前端严格采用 Apple-Class 风格，但优先服务运营效率。

关键词：

```text
克制
清晰
高密度但不拥挤
浅灰层级
Apple 蓝作为主要强调色
圆润但不过度
表格和筛选优先
```

不要做：

```text
营销式首页
大面积装饰图
花哨但无用的卡片
让运营看不懂的黑盒评分
把复杂内部条件塞进下拉框
```

## 前端交互规范

```text
筛选项要明确
下拉框质感要稳定、圆角、层级清楚
选中模块必须高亮
查询数据库时显示加载提示
返回页面时尽量保留筛选条件和页码
表格必须支持分页和页码跳转
收藏、状态、禁用词动作要即时反馈
```
